การเรียนรู้พื้นฐานไปสู่ระบบที่ซับซ้อน

(From Basic to Complex)

กลุ่มที่ 1: จุดเริ่มต้นของทุกสิ่ง

ก่อนจะมีโปรแกรม เราต้องเข้าใจก่อนว่าโปรแกรมทำหน้าที่จัดการกับอะไร (ข้อมูล -> กระบวนการ -> สารสนเทศ)

1. ข้อมูล (Data)

ข้อเท็จจริงดิบๆ ตัวเลข หรือตัวอักษรที่ยังไม่ได้ผ่านการประมวลผล

ตัวอย่าง: นาย ก., 37.5 (อุณหภูมิ), 120/80 (ความดัน)

2. กระบวนการ (Process)

ขั้นตอนการนำข้อมูลมาคำนวณ จัดกลุ่ม เปรียบเทียบ หรือวิเคราะห์

ตัวอย่าง: ระบบนำอุณหภูมิ 37.5 ไปเปรียบเทียบกับเกณฑ์มาตรฐาน

3. สารสนเทศ (Information)

ผลลัพธ์จากการประมวลผลที่นำไปใช้ประโยชน์หรือตัดสินใจได้ทันที

ตัวอย่าง: รายงานเตือน "พบผู้ป่วยความเสี่ยงติดเชื้อ 5 ราย"

วิธีเปรียบเทียบ (Chef Metaphor)

  • Data = วัตถุดิบดิบ (ผัก หมู เครื่องปรุง)
  • Process = ขั้นตอนการผัด การต้ม การปรุงรส
  • Information = อาหารที่ปรุงสุกพร้อมทาน (ต้มยำกุ้ง)
  • Software = "พ่อครัว" ที่ลงมือทำตามสูตรอาหาร

วงจรการประมวลผลสารสนเทศ (IPO-S-F Cycle)

เส้นทางการไหลของข้อมูลแบบครบวงจร ตั้งแต่ป้อนเข้าสู่ระบบไปจนถึงการนำผลลัพธ์มาใช้

1. Input

รับข้อมูลเข้า

นำข้อมูลดิบเข้าสู่ระบบ

ตัวอย่าง: พยาบาลพิมพ์ค่าความดัน (160/95) เข้าสู่ระบบคอมพิวเตอร์

2. Process

ประมวลผล

สมองกลคิดคำนวณ เปรียบเทียบ

ตัวอย่าง: ระบบนำ 160/95 ไปเทียบเกณฑ์มาตรฐานของร่างกาย

3. Storage

จัดเก็บข้อมูล

บันทึกเพื่อใช้ในอนาคต

ตัวอย่าง: บันทึกประวัติความดันลงในฐานข้อมูล (DBMS) กลาง

4. Output

แสดงผลลัพธ์

นำเสนอสารสนเทศที่ประมวลผลแล้ว

ตัวอย่าง: แจ้งเตือนสีแดงหน้าจอคอมพิวเตอร์หมอ "ความดันสูงวิกฤต!"

5. Feedback

ข้อมูลป้อนกลับ

นำผลลัพธ์ไปประเมิน และวนกลับไปเริ่มต้นใหม่

ตัวอย่าง: หมอเห็นแจ้งเตือน 👉 สั่งยาลดความดัน 👉 การสั่งยานี้จะถูกคีย์เป็น Input ใหม่
กระบวนการวนซ้ำต่อเนื่อง (Feedback Loop)

กลุ่มที่ 4: ภาพรวมระดับองค์กร

เมื่อองค์กรใหญ่ขึ้น แอปพลิเคชันและข้อมูลมหาศาลจะต้องถูกนำมาประกอบร่างกันเป็น "ระบบนิเวศ" (Ecosystem)

ระบบจัดการฐานข้อมูล (DBMS)

เปรียบเสมือน "ผู้จัดการห้องสมุดอัจฉริยะ" หรือ "ตู้เซฟกลาง" ขององค์กร

ไม่ใช่แค่พื้นที่เก็บไฟล์ธรรมดา แต่เป็นตัวกลางที่คอยบริหารจัดการข้อมูลหลังบ้านทั้งหมด เพื่อให้มั่นใจว่าข้อมูลถูกต้องและปลอดภัย

ศูนย์รวม (Centralized)

ข้อมูลเก็บที่เดียว ทุกแผนกดึงข้อมูลชุดเดียวกันไปใช้ ป้องกันการทำงานซ้ำซ้อน

ความถูกต้องตรงกัน

หากผู้ป่วยเปลี่ยนเบอร์โทร แก้ไขแค่จุดเดียว ระบบอื่นๆ จะอัปเดตตรงกันทั้งโรงพยาบาล

การกำหนดสิทธิ์ (Security)

รปภ.ดิจิทัล ควบคุมว่าใครมีสิทธิ์ดู หรือแก้ไขข้อมูลส่วนไหนได้บ้าง

ตัวอย่างที่เห็นภาพ: หากไม่มี DBMS ประวัติแพ้ยาของนาย ก. อาจอยู่ที่ห้องยา แต่แผนกฉุกเฉินไม่รู้ ทำให้เกิดอันตรายได้ แต่เมื่อมี DBMS (เช่น MySQL, SQL Server) ข้อมูลนี้จะถูกลิงก์ถึงกันหมดทันที

ระบบสารสนเทศ (Information System: IS)

เปรียบเสมือน "วงออร์เคสตรา" ที่รวมทุกองค์ประกอบเข้าด้วยกันเพื่อให้บรรลุเป้าหมายองค์กร

คนส่วนใหญ่มักเข้าใจผิดว่าระบบสารสนเทศมีแค่ "คอมพิวเตอร์" และ "โปรแกรม" แต่ในความเป็นจริง ระบบจะทำงานไม่ได้เลยหากขาด "คน" และ "กระบวนการทำงาน"

5 เสาหลักของระบบสารสนเทศ (5 Pillars of IS):

1. Hardware

อุปกรณ์ (คอมฯ, แท็บเล็ต)

2. Software

โปรแกรม (Web App, HIS)

3. Data

ข้อมูลที่จะป้อนเข้าสู่ระบบ

4. People

บุคลากรผู้ใช้งาน

5. Process

ขั้นตอนการทำงาน/กฎ

ตัวอย่างที่เห็นภาพ: ระบบ HIS ของโรงพยาบาลจะล้มเหลวทันที ต่อให้โปรแกรม (Software) ซื้อมาแพงและดีแค่ไหน ถ้าพยาบาลหรือหมอ (People) ไม่เข้าใจวิธีใช้ หรือไม่ยอมทำตามขั้นตอน (Process) ในการบันทึกข้อมูลเข้าสู่ระบบ

กรณีศึกษา: วิเคราะห์และออกแบบระบบ (IPO Model)

ตัวอย่างการนำวงจร IPO มาวิเคราะห์การสร้าง "ระบบคำนวณค่าดัชนีมวลกาย (BMI) สำหรับคนเอเชีย"

การวิเคราะห์ระบบ BMI (Asian Criteria)

1. Input

ข้อมูลที่ต้องรับเข้า

  • น้ำหนัก: กิโลกรัม (kg)
  • ส่วนสูง: เซนติเมตร (cm) หรือ เมตร (m)

2. Process

ขั้นตอนประมวลผล

  1. แปลงส่วนสูงให้เป็น "เมตร" (หาร 100)
  2. คำนวณสูตร:
    BMI = น้ำหนัก(kg) / [ส่วนสูง(m)]²
  3. เปรียบเทียบเกณฑ์ (คนเอเชีย):
    • • < 18.5 : น้ำหนักน้อย
    • • 18.5 - 22.9 : ปกติ (สุขภาพดี)
    • • 23.0 - 24.9 : ท้วม / โรคอ้วนระดับ 1
    • • 25.0 - 29.9 : อ้วน / โรคอ้วนระดับ 2
    • • >= 30.0 : อ้วนมาก / โรคอ้วนระดับ 3

3. Output

สิ่งที่ระบบแสดงผล

  • ตัวเลข BMI: เช่น 24.2
  • ผลการประเมิน: ท้วม / โรคอ้วนระดับ 1
  • คำแนะนำ: ควรควบคุมอาหารและออกกำลังกาย
Assignment / งานเดี่ยว

ใบงาน: การวิเคราะห์และออกแบบระบบคำนวณสถิติสาธารณสุข

ให้นักศึกษาวิเคราะห์การทำงานของระบบตาม IPO Model (Input - Process - Output)
พร้อมระบุสูตรการคำนวณให้ชัดเจน โดยรับผิดชอบหัวข้อตามหมายเลขลงท้ายของเลขที่ตนเอง (1-10)

01

เลขที่ลงท้ายด้วย 1

จำนวนผู้ป่วยนอกเฉลี่ยต่อวัน

เลขที่ลงท้ายด้วย 2

จำนวนผู้ป่วยในเฉลี่ยต่อวัน
